# 일일보고 작성기

개발자를 위한 일일 업무 보고서 자동 생성 도구입니다.

## 🌟 주요 기능

### 📝 보고서 작성
- **개발 단위 작업 관리**: 각 개발(스프린트) 작업을 잡(Job) 번호로 그룹화
- **자동 시간 계산**: 같은 잡 번호의 작업 시간이 자동으로 합산
- **작업 상태 관리**: 진행중, 완료, 배포완료, 배포대기, 테스트, 진행예정, 중단, 폐기

### 💾 로컬 저장소
- **잡 히스토리 자동 저장**: 한번 사용한 잡 번호는 localStorage에 저장
- **사용자 정보 저장**: 작성자 이름, 누적 시간, 최초 사용일
- **자동완성 지원**: 이전에 사용한 잡 번호 목록에서 선택 가능

### ⚡ 편의 기능
- **날짜별 자동 병합**: 같은 날짜의 보고서가 자동으로 합쳐짐
- **작업 기간 경고**: 2일 이상 진행되는 작업에 대한 자동 알림
- **누적시간 요약**: 모든 잡의 누적시간을 한눈에 확인
- **시간 수정 기능**: 잘못된 누적시간을 직접 수정 가능
- **연차/휴가 입력**: 간편한 연차 보고서 작성

### 📋 보고서 출력
- **템플릿 형식**: 회사 표준 양식에 맞는 보고서 생성
- **클립보드 복사**: 원클릭으로 보고서 복사
- **자동 날짜 증가**: 보고서 추가 후 다음 날로 자동 이동

## 🚀 사용 방법

### 온라인에서 바로 사용
GitHub Pages를 통해 바로 사용할 수 있습니다:
👉 [일일보고 작성기 사용하기](https://your-username.github.io/daily-report/)

### 로컬에서 사용
1. `daily_report_generator.html` 파일을 다운로드
2. 웹 브라우저에서 파일 열기
3. 바로 사용 가능 (별도 설치 불필요)

## 📖 사용 가이드

### 1. 기본 정보 입력
```
1. 작성자 이름 입력 (처음 한번만)
2. 보고 날짜 선택
```

### 2. 개발 작업 추가
```
1. "개발 추가" 버튼 클릭
2. 잡 번호 입력 또는 기존 잡 선택
3. 잡명, 개발명 입력
4. 작업 시간 입력 (시간/분)
5. 진행 상태, 시작일자 선택
6. 필요시 메모 작성
```

### 3. 보고서 생성
```
1. "보고서에 추가" 버튼 클릭
2. 오른쪽 패널에서 생성된 보고서 확인
3. "복사하기" 버튼으로 클립보드에 복사
```

### 4. 누적시간 관리
```
1. 보고서 상단의 "잡 누적시간 요약" 확인
2. "수정" 버튼으로 시간 조정 가능
3. "삭제" 버튼으로 히스토리에서 제거
```

## 🛠️ 기술 스택

- **Pure HTML/CSS/JavaScript**: 별도 프레임워크 없이 구현
- **localStorage**: 브라우저 로컬 저장소 활용
- **Responsive Design**: 모바일/데스크톱 대응

## 💡 특징

- ✅ **설치 불필요**: 웹 브라우저만 있으면 사용 가능
- ✅ **오프라인 작동**: 인터넷 연결 없이도 사용 가능
- ✅ **데이터 보안**: 모든 데이터가 사용자의 브라우저에만 저장
- ✅ **빠른 속도**: 서버 통신 없이 즉시 반응
- ✅ **간편한 백업**: HTML 파일 하나로 완결
